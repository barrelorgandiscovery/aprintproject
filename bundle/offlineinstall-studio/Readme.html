Nouveautés

2018 - Mai : Amélioration sur le Model editor, pour simplifier l'usage.

2016 - Octobre : Amélioration ergonomiques (sauvegarde de préférences utilisateur) mise en ligne de l'aide d'abord en français, avec une intégration d'un navigateur internet Passage à markdown pour les pages.

2016 - Aout : Passage à Java 8 (bug fix pour swing)

2016 - Ajout d'une extension de perforation avec GRBL Punch permet l'utilisation et la perforation de carton sur marchine Arduino ayant le logiciel GRBL Punch.

fin 2014 - Debut 2015 : amelioration du rendu sonore , ajout d'une envelope de volume pour l'attaque (plus beau pour les instruments de foire). Beaucoup d'amélioration visuelle et d'utilisation. Attention, il faut supprimer le cache de sound font pour avoir le nouveau rendu SF2.

Juin 2014 : Ajout d'outil pour le montage de carton, amélioration de performance sur le jeu (le démarrage du jeu du carton est plus rapide). Ajout d'un transactionnel sur le contenu du carton, pour permettre le rafraichissement des autres panneaux + taille du carton ... etc ...

Mars 2014 : Fix sur l'outil de sélection. Ameliorations sur le panneau de présentation des instruments. Nouveau panneau d'import de fichier midi ou Kar.

Fevrier 2014 : Amelioration du panneau de saisie de paramètres en groovy, ajout de la possibilité de mettre une librairie .jar disponible pour les scripts dans le répertoire aprintstudio. Cela permet d'utiliser d'autres code java dans le développement de scripts. (export DXF par exemple).

Ajout de deux boutons "basiques", measure et pan. ça aidera les débutants :-)

Janvier 2014 : Beaucoup d'améliorations vis à vis des possibilités de transformation de cartons ou de manipulation de fichiers midi par script, une première DSL est mise en place pour faciliter grandement la transformation de fichier midi, ou conversion vers book. quelques lignes de script suffisent maintenant pour effectuer des conversions ou des modifications. Toutes les modifications n'ont pas forcement ete consignées, mais le resultat sera surement présenté sous forme de script ou tutoriels.

Janvier 2013 : Ajout des mesures dans l'impression de ponçifs.

Juillet 2012 : Simplification de la gestion des instruments , un nouveau panneau beaucoup plus ergonomique. Ajout d'un affichage de barres d'outils dans la vue carton (simplifier la vue).Acces pour les extension à la liste des problemes, cela permet de proposer des extensions permettant d'aller plus loin dans le controle d'erreurs. Egalement la possibilité pour les extensions de réaliser des "toolwindow", fenetres dockables sur les cotés pour la vue carton. Ajout de composant de sélection simple d'instrument. ajout d'un bibliotheque de composants visuels, pour aller plus loin. Fix sur un bug dans le cadre de développement d'extension (lecture par fichier du fichier .extension.properties). Passage vers de multiples fenêtres pour les utilisateurs de MacOS. Les extensions peuvent maintenant proposer des librairies natives, ceci a été testé sur l'accès à des bibliothèques de port série.

Avril 2012: Ajout des mesures dans le logiciel, changements dans le format de stockage des cartons virtuels pour le stockage des annotations ou des changements de tempos Amelioration de la localisation sur le carton avec le CTRL+Roulette qui centre sur l'endroit pointé. Fix sur l'outil de modification de trous.

Mars 2012 : Documentation, Documentation, Documentation ... Quelques amélioration sur l'utilisation + Bug Fixs. Tentative de mettre en place une restitution sonore particulière pour les percussions. (encore qq pbs de restitution à voir). Ajout des erreurs de lecture MIDI dans la liste des erreurs. Améliorations Ergonomiques. Ameliorations significatives sur l'éditeur d'échantillons (wav)

Fevrier 2012 :Ajout du support d'un système de pilotage non midi (par prise série ou autre). Point d'extension, permettant à un tier de développer son système de pilotage pour son instrument. Bug fix sur le renommage de fichier pour les machintosh (les fichiers temporaires sont sur un autre filsystem). Ajout support drag and drop pour les quickscripts.

Decembre 2011 : Bug fixes, Ajout du drag and drop pour les fichiers book et instrumentbundle. Amélioration sur le feed back d'indexation et de recherche + ouverture des fichiers book. Augmentation du nombre de canaux utilisables directement (+de 16) et jusqu'à 64. Les instruments à plus de 10 registres sont maintenant restituables.

Novembre 2011 : Bug fixes, adaptation de quelques objets pour étendre l'utilisation des extensions

Septembre 2011 : Amelioration Qualité, (les utilisateurs aiment bien :-)

Juillet 2011, Ajout de la sauvegarde et de la définition de la couverture du carton

Bug Fixs, grandes amélioration dans la capacité de réalisation d'instruments très réalistes dans le jeu (ajout des percussions dans la description de l'instrument). amélioration de la documentation, plusieurs articles et rubriques ont été modifiés et ajoutée.

Avril 2011 : Bug Fixs, ajout d'un bouton de revalidation des erreurs dans la vue carton. Ajout des documentation groovy dans l'installation et la documentation en ligne. Ajout d'un bouton dans l'editeur de gamme pour ajouter des notes en série. Correction d'un bug de jeu lorsqu'il y a beaucoup de sons associé à l'instument. on peut maintenant sereinement mettre un son par tuyau :-)

Mars 2011 : Bug Fixs, corrections diverses d'olivier (pas encore fini :-)). Amélioration sur le support Midi dans le jeu. Ajout support fenetres flottantes/dockable pour simplifier l'ecran.

Ajout de deux fonction dans l'éditeur de gamme pour ajouter et supprimer une piste. Bug Fix sur la sauvegarde au format texte des erreurs.

Janvier 2011: bug fix sur l'arret du jeu dans la vue carton, fix sur les percussion dans la création de video
Inclusion des sources du programme dans le fichier .jar, modification de l'installation pour proposer la documentation developpeur.
