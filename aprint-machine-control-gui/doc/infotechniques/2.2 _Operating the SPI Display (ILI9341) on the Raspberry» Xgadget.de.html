2.2 "SPI display (ILI9341) on the Raspberry

So, in addition to simple status indicators, we can even play GameBoy games on the small SPI display. In the following instructions we show you both the connection and the initial configuration of the mini display on the Raspberry Pi.

Connect the display to the Raspberry

First, we have to connect the display to our Raspberry Pi, for a better overview, you will find below again the exact diagram before.

We connect the first SDO / MISO pin of our display to the MISO pin (GPIO9 - pin 21) of our Raspberry, for the second LED pin of the display we also need a 56 Ohm resistor - this will be connected to the GPIO18 (pin 12 ) of the Pi (because it also supports PWM, we can later adjust the brightness of the backlight).

Now we come to the third display pin "SCK", this we connect to the SCLK pin (on pin 23) of our craft computer. Now follows the "SDI / MOSI" connection of the SPI screen, this we connect to the MOSI (GPIO 10 - pin 19) of the mini PC. It continues with the "DC / RS" pin, which is connected to GPIO 24 (pin 18), furthermore we have to connect the RESET pin to the GPIO 25 (pin 22).

The third last "CS" display pin we need to connect to the GPIO 8 - this is pin 24 of the Raspberry's. The GND pin, however, we can connect to any ground port (pin 6, 9, 14, 20, 25, 30, 34, 39) of our Raspberry. Finally, we also need the VCC pin for the power supply, which we connect to one of the two existing 3.3 volt ports (pin 1 or 17).

functional test

After everything was properly connected we can connect our Raspberry Pi to the power supply. We perform our first functional test with the current Raspbian Jessie, you can download the corresponding image from the official website HERE .

First, if we have not already done so, we need to enable our SPI port in the operating system. To do this we execute the following command to enter the configuration menu:

sudo raspi-config

Now switch to the "Advanced Options" submenu and select "SPI" - this activates you and then closes the configuration assistant. Then we will update our system with the following command (this may take several minutes):

sudo rpi-update

Once the system has been updated successfully, you still have to restart the system with the following reboot command:

sudo reboot

Now re-connect with your Raspberry Pi, we'll do a first bump test with the following commands:

sudo modprobe fbtft_device name=tm022hdh26 rotate=90 speed=80000000 fps=60
sudo FRAMEBUFFER=/dev/fb1 startx
